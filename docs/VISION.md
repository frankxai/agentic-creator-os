# ACOS Vision

## The One-Liner

**The creator's studio, not the marketer's office.**

Every big agent framework is a dev-workflow OS. Office suites now ship free
AI plugins for generic knowledge work. ACOS is the only open system focused on
**creator production**: shipping actual content, video, music, visual assets,
and brand systems — with the creator's own identity driving every agent.

## Beliefs

1. **Creators should own their system.** Not rent a black-box SaaS. MIT core,
   local files, portable markdown. The test for every feature: *does this help
   someone build their own system, not just use someone else's?*
2. **Identity is the moat — the creator's, not ours.** Agents that write in a
   generic voice produce generic output. CREATOR.md makes identity a
   first-class, swappable contract.
3. **Production beats advice.** Skills that ship a track, a video, an article
   with hero assets — not skills that talk about shipping.
4. **Honesty compounds.** Generated stats, CI-enforced claims, telemetry-only
   hooks. An OS that lies about what it does teaches its agents to lie too.
5. **Compose, don't vendor.** Absorb patterns with attribution; integrate
   optionally; never fork someone else's system into the tree.

## Shape of Success (12 months)

- A stranger installs in 30 seconds, fills CREATOR.md, and ships their first
  branded multi-asset piece within an hour.
- The catalog is smaller and sharper than v11 — every skill maintained, every
  rule live, zero phantom claims.
- A community contributes creator-lane skills because the contract is clear.
- The open core funds itself: the pro layer (done-for-you identity setup,
  premium playbooks, hosted scheduled agents) converts a fraction of installs
  at [frankx.ai/acos](https://frankx.ai/acos).

## Non-Goals

- Being a general dev-agent framework (gstack, SDK harnesses do this)
- Feature-gating the core (the OS stays fully usable, free, forever)
- Chasing every model release into the tree (connectors absorb that churn)
