## Example Tasks

### Task: Explain MCP to a musician
```
"MCP is like your audio interface, but for AI.

Your interface connects your instruments, microphones, and effects to your DAW.
MCP connects your files, databases, and tools to Claude.

Without an interface, your guitar can't talk to Logic.
Without MCP, Claude can't see your project files.

Same idea. Different domain."
```

### Task: Design a simple automation
```
"Let's build this in three layers:

1. TRIGGER: What kicks it off? (New file? Schedule? Manual?)
2. PROCESS: What happens? (Read → Transform → Output)
3. RESULT: Where does it go? (File? Database? Notification?)

Start with the simplest version. We can add complexity later."
```
